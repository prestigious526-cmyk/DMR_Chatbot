BRIEF TECHNICAL SPECIFICATIONS

INTRODUCTION

The Secure HHPC is a Handheld Radio set for secure mode of voice communication in VHF band along with short data messages. It is digital protocol radio with built in encryption card for high end secrecy requirements. The Radio set operates in the frequency band of 136 MHz to 174 MHz  at 12.5 kHz channel spacing in secure DMR mode and at 25 kHz channel spacing in analog FM mode(in admin mode). The Preset channels and functions are selectable with rotary switches. The radio functions are set with the alphanumeric keypad on the front panel and also configurable via a Customer Programming Software (CPS). Two different power outputs viz. high and low are selectable through the front panel and CPS channel wise. It is capable of providing line of sight (LOS) communication in clear and secure mode along with short data messages. The battery operated radio is compatible with legacy radio in clear FM wideband mode and complies ETSI standard for DMR protocol. Analog communication is available in password protected admin mode.

RADIO FEATURES

The following features are available in Secure HHPC radio.

HHPC Remote Monitor:

Remote monitor feature allows a remote user to activate a target (misplaced / lost) HHPC’s microphone and transmitter for a period of time to assist in accessing the communication transpiring on lost/misplaced handsets.

HHPC Late Entry:

HHPC allows late entry call support in DMR mode.

HHPC Remote Enable/ Disable:

Remote Disable/Enable feature to stop any inappropriate use of HHPC or to prevent a stolen HHPC from functioning.

HHPC Emergency call:

Emergency call (short duration high power transmission) for sending notification to personnel at site   to convey message with respect to any crisis situation.

HHPC call types:

Type of calls: Individual, group and broadcast call facility is available.

HHPC data messaging

HHPC has facility of SMS texting in DMR mode.

Encryption:

HHPC has provision for SAG integration graded algorithm/encryption module for providing supplementary encryption.

Lone Worker Mode:

The DMR allows user to carry out automatic periodic check on personnel work alone at site.

Man-down feature:

The HHPC has special motion sensors to detect man down positions of an individual.

GPS tracker:

The HHPC Radio has inbuilt GPS tracker with 10m accuracy for better fleet management. The HHPC has feature of asking for GPS location from another HHPC and responding to this request with location coordinates.

MMI display:

The HHPC Radio has LCD with standard keypad. It displays GPS coordinates, battery strength indicator, caller ID, channel numbers etc.

Scanning on priority / Emergency call:

HHPC has a feature to automatically scan each channel in-turn and alert the user on active channel.

Fill Gun Interface:

The Radio Set receives the secure key data from Fill Gun when the Fill Gun is connected to the Radio Set and the “LOAD” key of the Fill Gun is pressed. This facility is password protected.

PC Interface:

The Radio Set receives the AES key data and preset frequency table from PC when the Radio Set is connected to COM port of PC loaded with appropriate PC to radio(CPS)  software.

VSWR protection at the ANTENNA socket

RF power output shall be as per para 5.1.2.1 under standard test conditions, after     subjecting the radio set to open circuit at the antenna port for short PTT pressing.

Reverse polarity protection

There is no permanent damage to radio on reverse polarity of power connections.

Backlit illumination of keypad

The facility of backlit illumination is available.

Keypad locking

The facility of keypad locking is available in the HHPC.

DCDM Mode

The HHPC allows two slot operation on single channel.

Digital Sensitivity:

The digital Sensitivity shall be minimum -117dbm for 5% BER.

Remote Kill:

The HHPC has a feature of remotely killing another HHPC. Remote killing involves algo erase and key erase.

Battery Life

HHPC should have battery life of 11 Hrs in custom digital mode and 9 Hrs in Analog Mode in 1:1: 18 ratio of HHPC operation (Tx: Rx: Standby) and is to be checked in 1 set per lot.

Spurious response rejection

The spurious response rejection should be 60dB or better.

Interoperability test

HHPC should be interoperable with the existing IAF HHPC in clear FM wideband mode.

Voice activation (Vox) test

HHPC transmit mode should be operated by voice in vox mode.

Functionality wise, on the transmit path, the radio receives clear voice from mic input (internal/external) through Codec. Data is processed in the required format. In order to initiate the communication in DMR clear mode or DMR custom mode, the controller checks for the presence of the encryption module PCB. In case the handshake is successful, the call/message is initiated. The custom channel is configured only after handshake. For analog wideband mode communication, the user should first go to admin mode. Then, the communication is possible in analog wideband mode only if the handshake with encryption module PCB is successful, i.e. after checking the presence of encryption module PCB. In custom encryption mode, the voice/data is passed on to the crypto card for encryption. In clear mode, data is passed onto the encryption module PCB and then modulated using TX up-converter IC and is sent to Power Amplifier card for further transmission. For SMS, the data is transmitted instead of voice in a similar manner. The clear analog wideband communication is plain FM communication in the range of 136-174MHz.

On the receive path, it receives the amplified signal from RF (PA) card and is down-converted to baseband level using down converter IC. Then this data is passed onto the DMR processor for audio level conversion. Codec is also used in this path. In custom encryption mode, this data is decrypted using crypto card before getting passed onto mic for audio output.

Baseband Card

The details of major modules of the system are as follows:-

1. Baseband card:

The battery mating connector, directly mounted on the baseband card received +7.4V (nom) from battery. The baseband card contains DC-DC converters for converting 7.4V into DC power supplies essential for operating all the modules. First U24 IC converts the incoming supply from battery (7.5V nominal) into 5V and 3.6V. These supplies in turn are converted into various supplies used for powering the analog and digital sections of the PCB. The digital section consists of Controller and processor both programmed for functions related to radio functionality. Keypad along with backlit graphic LCD display provides a user interface to crypto product operator for MMI operations. The detachable encryption module holds the algorithm for custom encryption. The baseband card also contains relevant interface ICs for specific radio functions like GPS location information, man-down feature implementation. The encryption module is an essential component for all types of communication. In analog wide-band mode (25 kHz channel spacing), the presence of encryption module is checked through handshake by the controller present on baseband card on I2C interface. In case of DMR clear mode also, the data is passed onto the encryption module for communication, through no encryption is done.

2. Power Amplifier Card: The Power amplifier module has major functions. First, in receive mode, i.e. when PTT is not pressed, it accepts the receive signal, does harmonic filtering, and amplifies it using low noise amplifier (LNA) for onward processing at baseband card. Secondly, in Tx mode, when PTT is pressed, it amplifies the transmit signal received from BB card to required level, i.e. 5W in high power mode and 1W in low power mode. Thirdly, it extracts the GPS signal from the receive signal and sends to Base Band card for extracting GPS information required by the controller.

3. Crypto card: The Crypto card provides the data encryption/decryption function of the equipment. This crypto card is physically mounted as a detachable module on the base band module. The Clear data from controller is encrypted in the crypto card and then passed on to the network controller of the baseband card. The crypto module interfaces with the baseband module of HHPC and it is a add on module to radio. The crypto module is a FPGA based high performance card. This card supports I2C protocol for data communication and UART protocol for key and algorithm loading. Module contains SPI Flash, RTC, and Schmitt trigger based RNG circuit.

4. LCD Display

The display size is 1.77 inch and LCD type is TFT, Normally White, transmissive.

Module dimension is 34.0(W) mm x 45.83(H) mm x 2.65(D)mm. supply voltage for analog min is 2.5V and current is 0.9A.

5. Keypad

The set contains an 18 key keypad. It is connected to baseband card on a 14-pin connector. The keypad is implemented in the form of a 3*6 matrix. Backlit facility is also available on the keypad and is controlled through the controller. The keypad is used for entering user related data.

The block diagram of the baseband card is as follows:-

a) Controller

The microcontroller used is a 32-bit ARM Cortex-M4 based microcontrollers with Floating Point Unit (FPU) for embedded applications which include an ARM Cortex-M0 coprocessor, with 1 MB of flash and 136 kB of on-chip SRAM and 16 kB of EEPROM memory. This MCU operates at a 12.28 MHz clock and 3.3V DC.

The controller contains code for controlling all the interfaces, including taking user inputs via keypad and displaying status to user using display. It also connects to crypto card using I2C interface. It also interfaces with function specific ICs for GPS and man down feature implementation.

b) DMR Processor

The DMR processor is responsible for analog mode and digital protocol implementation. The DSP circuit consists of a DSP and processes the base band signal. The DSP operates on an external clock of 12.28 MHz . The I/O section operates at 3.3V and the core section operates at 1.2V. It is a low power high performance base band processor supporting DMR protocol. With an external codec (audio band ADC and DAC), it completes the entire physical layer and data link layer, and most of the call control layer processing of DMR protocol. In DMR mode, it complies with ETSI TS 102 361.  This processor carries out the following processes:

Digital processing

* 4Level FSK and Baseband filter processing
* Vo-coder processing between audio codec and modulation/ demodulation
* Data scrambling

It is also responsible for Pre-emphasis/De-emphasis, Compressor/Expander processing and DMR protocol implementation.

This manual deals with repairs, testing, alignment and the manufacturers specifications of certain components that may be required to be replaced after any repair, testing or replacement has been done.

1.1	GENERAL PRECAUTIONS

The Secure Handheld Radio is modular in construction. Each Module sits on guided rivet on chassis and polarized Female connector, so the complete subsystem can be assembled/ disassembled easily. The socket, connectors and other items have generally been so designed with polarized connection so no interchange is possible. The disconnected items such as wires etc. should be properly tagged / marked with identification mark to enable easy identification. The module should be tightened on chassis only after the proper mating with rivet spacers /Guides and respective mating connectors.

The ICs, discrete components, and other components used in the Secure Handheld Radio, require careful handling. Care should be taken while replacing these components and only a miniature soldering iron of 25 watts should be used for soldering. A high standard of soldering should be maintained. Wiring of the circuit should be fiddled as minimum as possible.

DMR CPS

DMR CPS (Digital Mobile Radio Customer Programming Software) is a software application used to configure and program DMR device. It allows users to set up radio parameters, channels, contacts, and features according to their operational needs. This Customer Programing Software has multiple tabs: General, Channel Configuration, Misc Features, and Contact Lists.  When CPS software is opened, it shows its login screen as follows. Enter the provided username and password.

General –

In general Tab there are following features:

Serial Port Section includes COM Port selection and Baud rate (i.e. 38400). Select the COM port where the programming cable of radio is connected.

Radio Information Section includes Radio name and Radio ID. Password Section includes Admin Password and Power on Password. This section sets the Power ON password and the Admin password of the radio.

Side Key Programming Section includes Side Key 1 and Side Key 2. Both key is configurable and options available is Power, Squelch, Backlight and Power Save. If we select Power, the key pressing toggles the output RF power between Low Power and High Power. Squelch option is available only in Analog Mode.

Tx Timeout Programming: The purpose of TOT is to prevent any user from occupying a channel for an extended period. If the preset time expires, the radio will automatically terminate transmission

Display Radio Parameters include to select the Radio ID and open the file data using file data button.

Bottom Left Buttons includes-

Save All- Saves all settings.

Send All- Send all configuration to the radio.

Read All- Reads existing settings from the radio.

Send Checksum- Likely verifies data integrity before sending.

VHF Channel Configuration-

In VHF Channel Configuration these are the following features:

Channel Type- All channels are set to Digital, indicating digital radio communication is being used.

Channel Name – Channels are named ch-0 to ch- 1520, providing an identifier for each.

Tx Frequency and Rx Frequency

Tx Power- Two power mode is available one is LOW transmission power and other is high transmission power.

Rx only- Disabled for all channels, meaning these channels can both transmit and receive. For receive only, we can enable this option.

Color Code- Set to 0 or 1 for channel separation.

Slot Option- Select different time slot for different channels. Like all channels are assigned to Slot 1, indicating they are using Time Slot 1 in DMR communications (TDMA-based).

Encryption Type- Set to CLEAR for unsecured communication and Set to AES/CUSTOM for secured communication.

Misc Features-

In Misc Features these are the following features:

DMR Feature- This section contains Digital Mobile Radio features, with checkboxes for enabling/disabling specific remote functions. The enabled options include: Remote Radio Check, Remote Radio Monitor, Remote Radio Disable, Remote Kill, GPS Tracking.

Timeout Programming Section- This section allows setting timers for various radio functionalities, like for radio monitor, lone worker and man down.

VoX Parameters- This section includes VoX sensitivity to determine how easily the radio activates with voice input. VoX Timeout defines the delay (in seconds) before VoX stops transmitting after voice input ends.

Messages- This section contains pre-programmed quick text messages that can be sent from the radio.

Contact Lists-

In Contact List these are the following features:

Group List- This section manages group contacts for which this radio is a part.It will be able to listen to all communications meant for these groups.

Contact List- This sections stores individual radio contacts.

Setup is made as per procedure explained in chapter-4.Turn ON set by toggle power switch provided on the front panel. To operate the equipment at AC supply press toggle switch in downward direction and to operate the equipment at direct DC supply press toggle switch in upward direction. Ensure that RF I/P is connected at RF input port at rear panel. Follow the procedure in following para.

As equipment is turned ON initialization takes place, during initialization SM BITE is tested during initialization, if SM BITE is Pass, “Connect ST” will be displayed. Now Connect Valid ST (Security Token), on connecting Valid ST the display will show “ST Connected ”> “Access granted ”. After authentication of Security Token, BITE will be conducted to check the health status of HD VLF HF Receiver. After successful Power ON BITE operation, press Enter key to go to Main Menu, Main Menu will open.
